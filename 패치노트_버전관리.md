# 🚀 지능형 이메일 비서 (Email Assistant) - 패치노트 및 버전 관리

이 문서는 시스템 설계자와 개발자가 어떻게 아이디어를 주고받으며 우리 비서를 진화시켜왔는지 한눈에 볼 수 있는 **개발 일지(Release Notes)** 문서입니다.

---

## 🟢 [v1.12.0] - AI 오류 재시도 대기열 시스템 (현재 최신버전!)
- **문제**: AI 12회 전부 실패 시 "직접 확인하라"는 오류 메시지를 바로 전송하고 끝내던 구조
- **해결**: `retry_queue.json` 재시도 대기열 도입. AI가 12회 모두 실패하면 텔레그램 알림 없이 조용히 대기열에 등록. 5분 후 다음 폴링 사이클에서 자동으로 12회 재시도. 성공 시 정상 요약 전송. 재시도도 실패 시 그때서야 최종 오류 알림 발송. 결과에 상관없이 대기열에서 삭제(폐기).
- **총 시도 횟수**: 최대 12회 (첫 시도 6회 + 5분 후 재시도 6회)

## 🔵 [v1.11.1] - AI 모델 자동 폴백 체인 (503 과부하 방어)
- **문제**: 구글 `gemini-3-flash-preview` 서버 과부하(503) 발생 시 3회 재시도 후 비상 안내문으로 대체되어 요약 자체가 실패하던 문제
- **해결**: 모델 폴백(Fallback) 체인 도입. `gemini-3-flash-preview` 3회 모두 실패하면, 자동으로 안정화 정식 모델인 `gemini-2.0-flash`로 전환하여 추가 3회 재시도. 총 6번의 기회를 확보하여 요약 성공률을 대폭 향상시켰습니다.

## 🔵 [v1.11.0] - AI 완전 위임형(Gemini-First) 아키텍처 전면 재설계
- **파이썬 핑퐁 탐지 로직 완전 폐기**: 기존에 파이썬이 `Re:`, `Fwd:` 등의 문자열을 직접 긁어내서 핑퐁을 판단하던 단순 매칭 로직을 전면 삭제했습니다.
- **제미나이 완전 위임 구조 이식**: 이제 파이썬은 이메일 원본과 `thread_memory.json` 장부 전체를 인덱스 번호를 붙여서 제미나이에게 그대로 던집니다. 제미나이가 내용을 직접 읽고 핑퐁인지, 몇 번째인지, 새로 요약할 내용이 무엇인지 모두 판단하여 반환하면, 파이썬은 그 결과를 그대로 받아 장부에 저장(서기 역할)합니다.
- **6개 카테고리 분류 폐기**: 불필요한 `[영업]`, `[무역]` 등의 분류 연산을 제거하고 오직 `알림` / `스킵` 2가지만 판단합니다.
- **스레드별 가비지 컬렉터**: 각 주제 방의 요약 기록은 최신 5개만 유지하고, 30일간 소식 없는 방은 자동 삭제됩니다.
- **버튼 텍스트 전면 순화**: `[🚫 블랙리스트 차단]` → `[🔇 이 발송자, 앞으로 요약 알림 받지 않기]` 로 변경하여 오해의 소지를 제거했습니다.

## 🔵 [v1.10.0] - AI 맞춤 기피 학습(Few-Shot Learning) 단추 추가
- **제미나이 자동 진화 시스템**: 텔레그램 알림창 하단에 `[👎 이런 류의 메일 내용 요약 제외 (AI 학습)]` 버튼을 신설했습니다.
- **취향 맞춤 요약 스킵**: 사용자가 싫어요(👎) 버튼을 누르면, 이메일의 고유한 패턴(제목+내용 특징)을 추출하여 파이썬이 제미나이 전두엽의 교과서(`user_preferences.json`)에 영구 박제시킵니다. 이후 유사한 형태의 이메일이 오면 AI가 "주인님이 싫어하던 거다!" 라며 알아서 기피 패턴임을 인식하고 스스로 요약을 스킵합니다.
- **용량 제한 해제**: 사용자님의 사업 특성을 고려하여 학습 노트의 기억 용량 한계를 무제한으로 개방해 두었습니다.

## 🔵 [v1.9.0] - 거머리 퇴치용 '스팸 블랙리스트(영구 차단) 버튼' 연동

## 🔵 [v1.8.2] - AI 예민성 조정(스킵 필터 하향) 대형사고 방어 핫픽스
- **업무 메일 스킵 방어막 구축**: 사용자 편의를 위해 단답형을 솎아내려던 '스킵' 필터가 과도하게 예민하여, 본문이 짧은 중요한 업무 포워딩(FW)이나 단답형 요약 지시 메일까지 무의미한 메일로 오인하고 잘라버리는 치명적인 설계 결함을 발견하여 즉시 수정했습니다.

- **베트남 오피스 맞춤형 설계**: 모든 글로벌 이메일의 수신 시간이 텔레그램으로 배달될 때, 무조건 사용자님이 현재 체류 중이신 베트남 현지 시간(GMT+7)으로 100% 자동 계산되어 "2026-03-28 11:46 (베트남 시간)" 형태로 깔끔하게 통일되도록 파싱 로직을 핵심 엔진에 주입했습니다!
- **`BeautifulSoup` 무거운 라이브러리 제거**: 백엔드 경량화 기조에 맞춰 이메일 파싱 모듈 내부의 HTML 추출 로직을 순수 정규식 구조로 최적화하여 연산 속도를 더욱 높였습니다.

## 🔵 [v1.6.0] - 지능형 핑퐁 스레드 구축 및 AI 문맥 인지(Context-Aware) 탑재
- **텔레그램 답장(Slide Reply) 완벽 연동**: 메일 흐름이 단절되지 않고 핑퐁될 때, 텔레그램 비서가 알아서 이전 대화 말풍선을 물고 늘어지는 시각적 `[답장]` 형태로 알림을 보내어 완벽한 과거 스레드 역추적이 가능해졌습니다. 또 알림 최상단에 `[핑퐁 N회차]`를 명시하여 대화 진척도를 바로 파악합니다!
- **파이썬 가위질(Regex)을 통한 API 비용 획기적 절약**: 메일 본문을 전체 다 뭉텅이로 넘기는 돈 낭비를 방지하고자, 사내 그룹웨어 고유의 헤더(일자, From, To) 및 전 세계 공통 구분선(Boundary)을 기계적으로 포착합니다. 과거 내역은 통쾌하게 싹둑 자르고 "오직 방금 새로 덧붙인(Delta) 텍스트"만 제미나이에게 던지도록 아키텍처를 최적화했습니다. (API 토큰 절약률 극대화!)
- **직전 요약본 주입 방식 (Context Aware)**: 편지를 싹둑 잘랐다고 앞뒤 맥락을 놓치지 않습니다. 0원짜리 텍스트 임시 장부(`thread_memory.json`)에 **직전 요약본 메모**를 기록해 두었다가, AI 서버 통신 시 "방금 온 새 편지 원문 + 이전 대화 요약문"을 합쳐서 브리핑합니다! 제미나이는 앞뒤 맥락을 100% 엮어가며 절대 뚱딴지같은 번역을 하지 않습니다.
- **5단위 망각 방지 트리거 및 3개월(90일) 타임리셋**: 핑퐁이 너무 길어지면 까먹는 인간의 한계를 대신 방어하기 위해, 누적 5회/10회차 단위로 '전체 맥락 브리핑' 모드가 자동 발동됩니다. 추가로 먼지가 쌓인 지 90일 만에 돌아온 답장은, 카운트를 무시하고 강제 리셋 후 다시 전체 요약을 가동하여 감을 살려줍니다!
